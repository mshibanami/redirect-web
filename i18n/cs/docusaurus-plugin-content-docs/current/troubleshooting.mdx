# Řešení potíží

Tato stránka uvádí běžné problémy a jejich řešení při používání Redirect Web.

<TOCInline toc={toc} />

## Přesměrovací pravidla nefungují {#rules-not-working}

Pokud vaše pravidla nefungují podle očekávání, zkontrolujte prosím následující:

- **Rozšíření prohlížeče je nainstalované a povolené pro správnou stránku:** Ujistěte se, že máte nainstalované rozšíření Redirect Web a že je v prohlížeči povoleno pro webovou stránku, ze které chcete přesměrovat. Pokud na zdrojové stránce můžete otevřít vyskakovací okno rozšíření takto, je rozšíření nainstalováno správně:
    <Image src={ExtensionPopup} alt="Redirect Web extension popup" width="260" hasShadow/>

- **Konfigurace pravidla je správná:** Pečlivě zkontrolujte svá přesměrovací pravidla v nativní aplikaci Redirect Web. Ujistěte se, že vzory zdrojových URL, cílové URL a případné podmínky jsou správně nastavené. Chcete‑li ověřit, zda je pravidlo platné, zadejte příklad URL do sekce Examples na obrazovce Úpravy pravidla. Pokud je pravidlo platné, u svého vstupu uvidíte očekávanou cílovou URL, například takto:
    <Image src={RuleEditExampleResult} alt="An example of rule edit screen showing the result of example URL" width="260" hasShadow/>

## Vyskakovací okno rozšíření zobrazuje chybu {#error-shown-in-extension-popup}

<Image src={ErrorOverlay} alt="Redirect Web error overlay" width="260" hasShadow hasRoundedCorners/>

> Aplikace Redirect Web není nainstalována nebo nefunguje správně.  
> Získejte prosím aplikaci z App Store.

Pokud při otevření vyskakovacího okna rozšíření vidíte výše uvedenou chybu, znamená to, že nativní aplikace Redirect Web není nainstalovaná nebo nefunguje správně.

Ujistěte se prosím, že máte nainstalovanou nejnovější verzi Redirect Web z [App Store](https://apps.apple.com/app/id1571283503).

Možná bude také třeba po instalaci nativní aplikace znovu spustit prohlížeč.

> [!NOTE]
> * K používání rozšíření prohlížeče je potřeba nainstalovat nativní aplikaci Redirect Web. Pouze rozšíření bez aplikace nebude fungovat.
> * Uživatelé Safari nemusí rozšíření instalovat samostatně, protože je součástí nativní aplikace.

## Původní stránka se krátce zobrazí před přesměrováním {#original-page-flash}

K tomu dochází, pokud používáte přesměrovací pravidla typu **Original**. Tento typ nabízí různé pokročilé funkce, ale může způsobit krátké zobrazení původní stránky, protože může povolit načtení původní stránky před provedením přesměrování.

Otevřete obrazovku Úpravy pravidla a změňte **Type** na **DNR**, abyste se tomuto problému vyhnuli.

<Image src={EditRuleDnr} alt="Redirect Web extension popup" height="260" hasShadow hasRoundedCorners/>

> [!NOTE]
> Ve Firefoxu by k tomuto problému nemělo docházet ani při typu Original, protože používá API WebRequest, které zabraňuje načtení původní stránky.

## V Safari se v okně Nastavení zobrazují chyby {#safari-style-loading-errors}

V okně Nastavení Safari se mohou zobrazit chybové zprávy podobné následujícím:

<Image src={PopupStyleLoadingError} alt="Screenshot of error messages in Safari's Settings window" height="360" hasShadow hasRoundedCorners/>

> * Nelze najít "popup.mobile.css" ve zdrojích rozšíření. Jedná se o neplatnou cestu.
> * Nelze najít "toolbar-button.mobile.css" ve zdrojích rozšíření. Jedná se o neplatnou cestu.

To neovlivňuje funkčnost Redirect Web. Můžete je bezpečně ignorovat.

Tyto chyby jsou způsobeny tím, že vyskakovací okno rozšíření se snaží načíst styly jak pro desktop, tak pro mobilní zařízení, přičemž lze načíst pouze tyto soubory v závislosti na tom, zda používáte macOS nebo iOS. V důsledku toho se jeden ze souborů vždy nezdaří načíst, což je očekávané chování.

Chápeme, že zobrazení chybových zpráv může být znepokojující, proto plánujeme tyto zprávy potlačit v budoucí aktualizaci. Podrobnosti najdete v [GitHub issue #96](https://github.com/mshibanami/redirect-web/issues/96). Omlouváme se za zmatek!

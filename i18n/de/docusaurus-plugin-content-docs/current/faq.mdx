# FAQ

## Warum sollte ich der Erweiterung erlauben, meine Informationen zu lesen/ändern?

Möglicherweise sehen Sie eine solche Meldung, wenn Sie Redirect Web einrichten:

<IfBrowserStore stores='appStore'>
    <Image src={PermissionAlertSafari} alt="An permission requiring alert" width="200" margin="0 0 16px 0" hasShadow={true} />

    > Diese Erweiterung kann Webseiten lesen und ändern sowie Ihren Browserverlauf auf dieser Webseite sehen. **Dies kann sensible Informationen wie Passwörter, Telefonnummern und Kreditkarten umfassen.**
</IfBrowserStore>

<IfBrowserStore stores='appStore' includes={false}>
    <Image src={PermissionAlertChrome} alt="An permission requiring alert" height="200" hasRoundedCorners={true} hasShadow={true} margin="0 0 16px 0" />

    > Sie kann:  
    > **Alle Ihre Daten** auf **allen Webseiten** lesen und ändern
</IfBrowserStore>

Diese Erweiterung benötigt Ihre Erlaubnis, um die Seitennavigation zu beobachten und Sie dorthin umzuleiten, wohin Sie springen möchten. Ihr Browser zeigt dies an, wenn die Erweiterung die entsprechenden Berechtigungen anfordert. Wenn Sie dies nicht zulassen, kann diese Erweiterung nicht funktionieren.

Erteilte Berechtigungen werden nur für App-Funktionen wie die Umleitung verwendet, in Übereinstimmung mit der **[Datenschutzerklärung](./privacy-policy)**.

## Sollte ich die App trotzdem herunterladen, nachdem ich die Erweiterung für Chrome, Firefox oder Edge installiert habe? \{#app-for-non-safari-browsers}

Ja. Sie müssen [die Redirect Web App aus dem App Store](https://apps.apple.com/app/id1571283503) herunterladen, unabhängig davon, welchen Browser Sie verwenden. Sie können die Erweiterung über den Erweiterungs-Store Ihres Browsers installieren, aber die App ist erforderlich, um Ihre Regeln, Ihren Kaufverlauf und andere Einstellungen geräteübergreifend über iCloud zu verwalten.

## Funktioniert Redirect Web unter Windows, Android, Linux oder anderen Plattformen? \{#non-apple-platforms}

Derzeit nicht. Redirect Web ist eng an das Apple-Ökosystem gebunden, einschließlich iCloud und Universal Purchase, was die Bereitstellung auf anderen Plattformen erschwert.

## Warum nicht [Redirector](https://github.com/einaregilsson/Redirector)?

Redirector ist immer noch eine großartige Umleitungs-Erweiterung. Sie ist völlig kostenlos, Open-Source und funktioniert auch auf Nicht-Apple-Plattformen. Aber Redirect Web bietet einige zusätzliche Funktionen, die Ihnen gefallen könnten:

*   **Detaillierte Regeleinstellungen:** Mehrere Ausschlussmuster, mehrere Prozessübereinstimmungen und Zeichenersetzungsfunktionen, die nur in Redirect Web verfügbar sind.
*   **In nativen Apps öffnen:** Auf dem Mac können Sie wählen, wo die Ziel-URL geöffnet werden soll. (Es ruft den Befehl `open` direkt auf. Keine Deep Links.)
*   **[DNR](https://blog.mozilla.org/addons/2023/05/17/declarativenetrequest-available-in-firefox/) Unterstützung:** Bietet blitzschnelle Umleitung.
*   **Zentralisierte Regelverwaltung:** Verwalten Sie Regeln für alle Browser an einem Ort über die native App.
*   **Manifest V3 & V2 Unterstützung:** Hilfreich, während V2 in Chrome ausläuft.

## Warum benötigt diese Erweiterung eine neue Berechtigung? \{#permission-alert-on-safari}

Möglicherweise sehen Sie diese Meldung in Safari, nachdem Sie Redirect Web aktualisiert haben:

<img src={require("/img/safari-additional-permission-alert.webp").default} alt="An alert of disabling the app" width="280" />

> „Redirect Web“ wurde aktualisiert und fordert zusätzliche Berechtigungen an. Zum Schutz Ihrer Privatsphäre wurde diese Erweiterung deaktiviert.
>
> Sie können diese Erweiterung im Bereich „Erweiterungen“ der Safari-Einstellungen wieder aktivieren.

Dies liegt an der Berechtigung [declarativeNetRequestWithHostAccess](https://developer.apple.com/documentation/safariservices/safari_web_extensions/blocking_content_with_your_safari_web_extension), die für den [DNR](./rule-settings#type)-Typ erforderlich ist, der in Version 7 eingeführt wurde. Bitte aktivieren Sie die Erweiterung erneut. Entschuldigen Sie die Unannehmlichkeiten.

Nein, sie ist immer kostenlos. Den Quellcode aller Inhalte finden Sie auch [hier](https://github.com/mshibanami/redirect-web/tree/main/library).

## Einmaliger Kauf oder Jahresabonnement: Was ist am besten?

Wir empfehlen den einmaligen Kauf, da der Preisunterschied nur 2,00 US-Dollar beträgt.
Der Hauptgrund, warum wir das Abonnement anbieten, ist, Ihnen zu ermöglichen, alle Funktionen durch die kostenlose Testphase auszuprobieren. Hier ist der erwartete Anwendungsfall:

1.  Abonnieren Sie das Jahresabonnement.
2.  Kündigen Sie das Abonnement sofort, indem Sie zur Einstellungen-App gehen, nach „Abonnements“ suchen, auf diese App tippen und „Abonnement kündigen“ auswählen. Dadurch können Sie die kostenlose Testphase ohne weitere Verpflichtungen nutzen.
3.  Genießen Sie 7 Tage lang vollen Zugriff auf alle Funktionen.
4.  Wenn Ihnen die Funktionen gefallen, entscheiden Sie sich für den einmaligen Kauf.

Natürlich ist nichts falsch daran, das Jahresabonnement fortzusetzen, wenn Sie dies bevorzugen!

## Fügt Redirect Web Werbung oder andere irrelevante Inhalte in Websites ein?

Nein, es werden keine solchen Inhalte eingefügt. Nur Code, der zur Bereitstellung der in dieser Dokumentation und auf der App-Seite im App Store beschriebenen Funktionen notwendig ist, wird in Websites unter Safari injiziert.

## Besteht die Möglichkeit, dass Redirect Web jemals an jemand anderen verkauft wird?

100% nein. Browser-Erweiterungen haben umfassende Kontrolle über Ihr Surfen und Ihre Daten, und eine Übergabe von Redirect Web an einen anderen Eigentümer würde die Tür für Datenschutzverletzungen öffnen. Wir haben gesehen, wie Erweiterungen den Besitzer gewechselt und dann ohne Vorwarnung Benutzerdaten gesammelt oder Werbung eingefügt haben. Das wird hier niemals passieren. Weitere Informationen zu diesen Risiken finden Sie in Matt Frisbies Artikel ["The Ugly Business of Monetizing Browser"](https://mattfrisbie.substack.com/p/the-ugly-business-of-monetizing-browser) und Brian Krebs' Untersuchung ["Is Your Browser Extension a Botnet Backdoor?"](https://krebsonsecurity.com/2021/03/is-your-browser-extension-a-botnet-backdoor/).

Redirect Web basiert auf Vertrauen und Verantwortung. Wir betrachten es als unsere Pflicht, Ihre Informationen sicher zu halten, und wir lehnen eine Eigentumsübertragung unter keinen Umständen ab. Redirect Web wird immer in unseren Händen bleiben, sodass Sie mit vollem Vertrauen surfen können.

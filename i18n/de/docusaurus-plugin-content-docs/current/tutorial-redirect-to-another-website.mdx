# Fall 1: Zu einer anderen Website weiterleiten

Angenommen, Sie sind süchtig nach Twitter, und um diese Gewohnheit zu durchbrechen, entscheiden Sie sich, zu [Unrush](https://unrush.org) zu gehen, um motivierende Zitate und eine kurze Meditation zu erhalten, wann immer Sie versehentlich Twitter öffnen. Lassen Sie uns eine Regel dafür erstellen!

### Schritt 1. Öffnen Sie den Bildschirm „Regel bearbeiten“

Öffnen wir den Bildschirm *Regel bearbeiten*, um mit der Erstellung Ihrer eigenen Regel in der App zu beginnen. Sie können dies tun, indem Sie auf die Plus-Schaltfläche (+) im Tab *Meine Regeln* klicken.

### Schritt 2. „Weiterleiten von“ einrichten

Zuerst sollten Sie den Bereich *Weiterleiten von* im Bildschirm *Regel bearbeiten* in der App einrichten. Die Ziele sind URLs, die mit `https://twitter.com/` beginnen.

In diesem Fall können Sie das folgende Muster im [Wildcard](/rule-settings#url-pattern-wildcard)-Modus angeben:

```
https://twitter.com/*
```

Im Wildcard-Modus bedeutet `*`, dass es auf alles passt (= null oder mehr Zeichen).

Aber Moment mal. `twitter.com` wurde in `x.com` umbenannt. Sie können einfach `twitter.com` durch `x.com` ersetzen, aber niemand weiß, wann der neue Besitzer seine Meinung wieder ändert und es rückgängig macht. Daher lassen Sie uns sowohl `twitter.com` als auch `x.com` anvisieren.

Wechseln Sie dazu von Wildcard zu [Regulärer Ausdruck](/rule-settings#url-pattern-regular-expression) und legen Sie dies als Muster fest:

```
https://(twitter|x).com/.*
```

* `(twitter|x)`: es zielt sowohl auf `twitter` als auch auf `x` ab. (`|` wird als *Pipe* bezeichnet.)
* `.*`: Es ist dasselbe wie Wildcard's `*`. Genauer gesagt bedeutet `.` *ein beliebiges Zeichen* und `*` bedeutet *alles vor diesem Symbol, das beliebig oft wiederholt wird*, was dazu führt, dass es auf alles passt.

Reguläre Ausdrücke sind etwas kompliziert, aber sobald man sich daran gewöhnt hat, werden sie ein mächtiges Werkzeug sein. Wir empfehlen [RegExr](https://regexr.com) als Spielwiese, um zu analysieren, wie Ihr reguläres Ausdrucksmuster funktioniert.

> [!TIP]
> In regulären Ausdrücken wird `.` in `(twitter|x).com` ebenfalls als *beliebiges Zeichen* behandelt. Daher passt `(twitter|x).com/.*` zum Beispiel auch auf `twitter1com/` oder `x_com/`.
>
> Um präziser zu sein, können Sie es in `(twitter|x)\.com/.*` ändern. `\` wird verwendet, um `.` als literalen Punkt zu behandeln und nicht als Sonderzeichen.
>
> Allerdings gibt es solche URLs im allgemeinen Internetumfeld nicht. Daher können Sie `.` als Sonderzeichen belassen, wenn Sie möchten.

### Schritt 3: „Weiterleiten nach“ einrichten

Geben Sie die URL einfach wie folgt an:

```
https://unrush.org/
```

### (Optional) Schritt 4: Regel prüfen

Sie können überprüfen, wie die Regel funktioniert, indem Sie eine Beispiel-URL im Abschnitt *Beispiele* im Bildschirm *Regel bearbeiten* hinzufügen. Wenn Sie zum Beispiel `https://x.com/UN` eingeben, wird `https://unrush.org/` als Ergebnis angezeigt.

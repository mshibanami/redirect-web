# Ρυθμίσεις Κανόνα

Αυτή η σελίδα παρέχει λεπτομερείς πληροφορίες για κάθε ρύθμιση στην οθόνη Επεξεργασία Κανόνα της εφαρμογής Redirect Web.

## Επιλογές

### Τύπος

Καθορίστε την επιλογή **Τύπος** για να ελέγξετε τον τρόπο με τον οποίο η εφαρμογή χειρίζεται την ανακατεύθυνση. Μπορείτε να επιλέξετε από τα εξής:

*   **Original** (Προεπιλογή)
    *   Αυτό χρησιμοποιεί παραδοσιακά Web APIs για τον έλεγχο της ανακατεύθυνσης. Επιπλέον, χρησιμοποιεί [το Tabs API](https://developer.mozilla.org/en-US/docs/Mozilla/Add-ons/WebExtensions/API/tabs) ως εφεδρική λύση.
        *   Στον Firefox, χρησιμοποιεί το [WebRequest API](https://developer.mozilla.org/docs/Mozilla/Add-ons/WebExtensions/API/webRequest) για τη διαχείριση της ανακατεύθυνσης.
    *   Μπορείτε να χρησιμοποιήσετε όλες τις επιλογές εκτός από [Τύπους Πόρων](#resource-types) και [Μεθόδους Αίτησης](#request-methods).
    *   Αυτό είναι πιο αργό από τον *Δηλωτικό* τύπο και μπορεί να προκαλέσει επιπλέον αιτήσεις δικτύου.
*   **DNR** (Πειραματικό για Safari):
    *   Αυτός ο τύπος λειτουργεί πολύ πιο γρήγορα από τον αρχικό τύπο, επειδή δεν ξεκινά αίτηση δικτύου για την αρχική διεύθυνση URL.
    *   Αυτό σας επιτρέπει να καθορίσετε [Τύπους Πόρων](#resource-types) και [Μεθόδους Αίτησης](#request-methods).
    *   ⚠️ Δεν μπορείτε να χρησιμοποιήσετε ορισμένες επιλογές, όπως [Επεξεργασία Ομάδας Σύλληψης](#capturing-group-processing) και [Εξαιρούμενα Πρότυπα URL](#excluded-url-patterns), καθώς δεν έχουν υποστηριχθεί ακόμη από το DNR API.
    *   ⚠️ Στο Safari, επί του παρόντος δεν μπορείτε να συμπεριλάβετε κάθετες γραμμές (`|`) στο μοτίβο της Κανονικής σας Έκφρασης. [Λεπτομέρειες](https://github.com/mshibanami/redirect-web/issues/44)
    *   ⚠️ Δεδομένου ότι το DNR API του Safari εξακολουθεί να έχει ορισμένα ζητήματα, θεωρούμε ότι παραμένει μια πειραματική λειτουργία για το Safari. Μπορείτε να βρείτε τη λίστα όλων των γνωστών ζητημάτων [εδώ](https://github.com/mshibanami/redirect-web/issues?q=state%3Aopen%20label%3ADNR%20label%3ASafari).

### Ανακατεύθυνση Από

Η επιλογή **Ανακατεύθυνση Από** σας επιτρέπει να καθορίσετε ένα μοτίβο URL ιστοσελίδων από τις οποίες θέλετε να κάνετε ανακατεύθυνση. Μπορείτε να επιλέξετε είτε [Wildcard](#wildcard) είτε [Κανονική Έκφραση](#regular-expression).

Για παράδειγμα, αν καθορίσετε `https://example.com/*` με Wildcard, ταιριάζει με `https://example.com/` ή `https://example.com/hello`.

> [!NOTE]
> Στην επιλογή Ανακατεύθυνση Προς, μπορείτε να αναφερθείτε σε ολόκληρη την αντιστοιχία χρησιμοποιώντας το `$0` ή σε μερικές αντιστοιχίες χρησιμοποιώντας τα `$1`, `$2`, ... Ελέγξτε τις λεπτομέρειες στο [Μοτίβο URL](#url-pattern) σε αυτή τη σελίδα.

#### Τύποι Πόρων

![Supported Types: DNR](https://img.shields.io/badge/Types-DNR-blue)

Η επιλογή **Τύποι Πόρων** σας επιτρέπει να καθορίσετε τις κατηγορίες αιτήσεων ιστού, όπως εικόνες, JavaScript και φύλλα στυλ, στις οποίες εφαρμόζεται ο κανόνας.
Για παράδειγμα, αν ορίσετε `script`, μπορείτε να ανακατευθύνετε τα αρχεία JavaScript που φορτώνονται από ιστοσελίδες.

Επί του παρόντος, είναι διαθέσιμα τα εξής:
`main_frame`, `sub_frame`, `stylesheet`, `script`, `image`, `font`, `xmlhttprequest`, `ping`, `media`, `websocket`, `other`

Η προεπιλεγμένη ρύθμιση είναι `main_frame`, η οποία είναι η σελίδα κορυφαίου επιπέδου που φορτώνεται σε μια καρτέλα.

Ελέγξτε τις λεπτομέρειες για κάθε τύπο πόρου στο [mdn web docs](https://developer.mozilla.org/en-US/docs/Mozilla/Add-ons/WebExtensions/API/declarativeNetRequest/ResourceType).

#### Μέθοδοι Αίτησης

![Supported Types: DNR](https://img.shields.io/badge/Types-DNR-blue)

Η επιλογή **Μέθοδοι Αίτησης** σας επιτρέπει να ορίσετε τις μεθόδους HTTP-στόχο της διεύθυνσης URL προέλευσης.

Όλες οι μέθοδοι ορίζονται από προεπιλογή.

Ελέγξτε τις λεπτομέρειες για κάθε μέθοδο στο [mdn web docs](https://developer.mozilla.org/en-US/docs/Web/HTTP/Methods).

> [!WARNING]
> Στο Safari, αυτή η επιλογή ενδέχεται να μην είναι διαθέσιμη επειδή η υλοποίηση της Apple για τον τύπο DNR δεν φαίνεται να την υποστηρίζει επί του παρόντος. Στείλαμε σχόλια στην Apple ως FB14502272.

#### Στοχευμένα Προγράμματα Περιήγησης

![Supported Platforms: macOS](https://img.shields.io/badge/Platforms-macOS-white)

Η επιλογή **Στοχευμένα Προγράμματα Περιήγησης** σας επιτρέπει να καθορίσετε σε ποιο πρόγραμμα περιήγησης πρέπει ή δεν πρέπει να εφαρμοστεί ο κανόνας.

*   Υπάρχουν 2 τύποι προγραμμάτων περιήγησης που μπορείτε να επιλέξετε:
    *   **Προγράμματα περιήγησης που μπορείτε να επιλέξετε από προεπιλογή**: Μπορείτε να επιλέξετε Safari, Chrome, Firefox, Edge, Opera, Arc, Brave και Vivaldi από προεπιλογή. Αυτό δεν διαφοροποιεί παραλλαγές του ίδιου προγράμματος περιήγησης, όπως το Safari και το Safari Technology Preview.
    *   **Προγράμματα περιήγησης που καθορίζονται από τον χρήστη**: Πατώντας το κουμπί Προσθήκη στο αναδυόμενο παράθυρο, μπορείτε να επιλέξετε οποιοδήποτε πρόγραμμα περιήγησης εκτός από το Safari ή το Safari Technology Preview που έχει εγκατεστημένη την επέκταση Redirect Web.
*   Το πλαίσιο ελέγχου **Όλα εκτός των επιλεγμένων** σας επιτρέπει να στοχεύετε κάθε πρόγραμμα περιήγησης εκτός από αυτά που έχετε επιλέξει.

> [!WARNING]
> Το Redirect Web δεν μπορεί να διαφοροποιήσει αν το πρόγραμμα περιήγησης είναι **Safari** ή **Safari Technology Preview**, επειδή η εφαρμογή ελέγχει τη γονική διαδικασία της επέκτασης για να το εντοπίσει, αλλά αυτά τα δύο προγράμματα περιήγησης μοιράζονται την ίδια γονική διαδικασία (`/sbin/launchd`).

### Ανακατεύθυνση Προς

Η επιλογή **Ανακατεύθυνση Προς** καθορίζει μια διεύθυνση URL προορισμού στην οποία θέλετε να ανακατευθύνετε, από την αρχική διεύθυνση URL που ταιριάζει με την επιλογή *Ανακατεύθυνση Από*. Μπορείτε επίσης να αντικαταστήσετε τις ομάδες σύλληψης με `$1`, `$2`, ... ή ολόκληρη την αντιστοιχία με `$0`. Αυτά σας βοηθούν να καθορίσετε δυναμικά τον προορισμό. Ελέγξτε τις λεπτομέρειες στο [Μοτίβο URL](#url-pattern) σε αυτή τη σελίδα.

Για παράδειγμα, αν ορίσετε τον ακόλουθο κανόνα:

*   **Ανακατεύθυνση Από**: `https://google.com/*` (Wildcard)
*   **Ανακατεύθυνση Προς**: `https://apple.com/$1`

και ταιριάζει με `https://google.com/hello`, τότε η διεύθυνση URL προορισμού θα είναι `https://apple.com/hello`.

Επιπλέον, μπορείτε να τροποποιήσετε τα κείμενα των `$1`, `$2`, ... πριν κάνετε αντικαταστάσεις. Ελέγξτε την ενότητα [Επεξεργασία Ομάδας Σύλληψης](#capturing-group-processing) για λεπτομέρειες.

> [!TIP]
> Μπορείτε να καθορίσετε ένα προσαρμοσμένο σχήμα URL για να ανοίξετε μια εφαρμογή. Αυτά είναι παραδείγματα εφαρμογών που υποστηρίζουν deep linking:
>
> *   Figma: `figma://file/Your_Figma_ID`
> *   Firefox: `firefox://open-url?url=https://example.com/hello`
> *   Google Chrome: `googlechromes://example.com`
> *   Microsoft Edge: `microsoft-edge://example.com`
> *   Notion: `notion://www.notion.so/Your_Note_ID`
> *   Slack: `slack://open`

#### Εφαρμογή

![Supported Types: Original](https://img.shields.io/badge/Types-Original-blue) ![Supported Platforms: macOS](https://img.shields.io/badge/Platforms-macOS-white)

Εάν θέλετε να καθορίσετε μια εφαρμογή στην οποία θέλετε να ανοίξει η διεύθυνση URL προορισμού, χρησιμοποιήστε το σύνθετο πλαίσιο **Εφαρμογή**. Αυτό είναι διαθέσιμο μόνο στο macOS.

> [!WARNING]
> Μπορείτε να ανοίξετε μόνο μια εφαρμογή που υποστηρίζει το [App Sandbox](https://developer.apple.com/documentation/security/app_sandbox). Επίσης, βεβαιωθείτε ότι η εφαρμογή υποστηρίζει το άνοιγμα της διεύθυνσης URL που θέλετε να ανοίξετε.

### Επεξεργασία Ομάδας Σύλληψης

![Supported Types: Original](https://img.shields.io/badge/Types-Original-blue)

Η επιλογή **Επεξεργασία Ομάδας Σύλληψης** σας επιτρέπει να καθορίσετε τον τρόπο επεξεργασίας των συλλαμβανόμενων ομάδων που μπορείτε να αντικαταστήσετε στην επιλογή *Ανακατεύθυνση Προς* με `$1`, `$2`...

Έτσι δημιουργούνται οι ομάδες σύλληψης:

*   **Wildcard**: Τα κείμενα που αντιστοιχούν σε `*` και `?` συλλαμβάνονται αυτόματα.
*   **Κανονική Έκφραση**: Τα κείμενα που αντιστοιχούν σε μέρος του μοτίβου εντός `()` συλλαμβάνονται.

Μπορείτε να επιλέξετε μία ή περισσότερες από τις ακόλουθες διαδικασίες:

*   **Κωδικοποίηση/Αποκωδικοποίηση URL**: Αυτό εφαρμόζει [κωδικοποίηση ποσοστού](https://en.wikipedia.org/wiki/Percent-encoding) ή αποκωδικοποίηση σε μια ομάδα σύλληψης. Για παράδειγμα, αν κωδικοποιήσετε το `https://example.com/hello`, μετατρέπεται σε `https%3A%2F%2Fexample.com%2Fhello`. Η αποκωδικοποίηση λειτουργεί με τον αντίθετο τρόπο.
*   **Κωδικοποίηση/Αποκωδικοποίηση Base64**: Αυτό αποκωδικοποιεί/κωδικοποιεί ένα κείμενο σε [Base64](https://en.wikipedia.org/wiki/Base64). Για παράδειγμα, μπορείτε να κωδικοποιήσετε το `hello` σε `aGVsbG8=`, και να το αποκωδικοποιήσετε ξανά σε `hello`.
*   **Αντικατάσταση Εμφανίσεων**: Αυτό αντικαθιστά έναν ή περισσότερους χαρακτήρες σε μια ομάδα, που αντιστοιχεί σε έναν **Στόχο**, με μια **Αντικατάσταση**. Για παράδειγμα, αν η *Ομάδα* είναι `hello` και ο *Στόχος* είναι `l` και η Αντικατάσταση είναι `y`, τότε τροποποιείται σε `heyyo`.

> [!NOTE]
> Αυτό αφορά την επεξεργασία κάθε ομάδας σύλληψης και δεν επηρεάζει ποιες διευθύνσεις URL εξαιρούνται από τα [Εξαιρούμενα Πρότυπα URL](#excluded-url-patterns). Για παράδειγμα, ας υποθέσουμε ότι έχετε αυτόν τον κανόνα:
>
> *   **Ανακατεύθυνση Από**: `https://example.com/(hello.*)`
> *   **Επεξεργασία Ομάδας Σύλληψης**:
>     *   Ομάδα: `$1`
>     *   Στόχος: `.*` (Κανονική Έκφραση)
>     *   Αντικατάσταση: `hello`
> *   **Εξαιρούμενο Μοτίβο URL**: `https://example.com/hello`
>
> Σε αυτή την περίπτωση, το `https://example.com/hello_world` δεν θα εξαιρεθεί, ενώ το `https://example.com/hello` θα εξαιρεθεί.

### Εξαιρούμενα Πρότυπα URL

![Supported Types: Original](https://img.shields.io/badge/Types-Original-blue)

Η επιλογή **Εξαιρούμενα Πρότυπα URL** σας επιτρέπει να καθορίσετε τις διευθύνσεις URL που δεν ανακατευθύνονται. Αυτό μπορεί να είναι χρήσιμο για να αποφύγετε βρόχους ανακατεύθυνσης ή να εξαιρέσετε ορισμένα μέρη μιας ιστοσελίδας από την ανακατεύθυνση.

Μπορείτε να καθορίσετε εξαιρούμενα μοτίβα URL χρησιμοποιώντας είτε τον τύπο μοτίβου Κανονικής Έκφρασης είτε Wildcard.

### Παραδείγματα

Η επιλογή **Παραδείγματα** σας επιτρέπει να δοκιμάσετε τον κανόνα ανακατεύθυνσης παρέχοντας δείγματα διευθύνσεων URL. Προσθέτοντας μια διεύθυνση URL δείγματος, μπορείτε να ελέγξετε αν ο κανόνας λειτουργεί όπως αναμένεται πριν τον εφαρμόσετε πραγματικά.

### Σχόλια

Η επιλογή **Σχόλια** σας επιτρέπει να προσθέσετε οποιεσδήποτε σημειώσεις ή σχόλια σχετικά με τον κανόνα ανακατεύθυνσης. Αυτό μπορεί να είναι χρήσιμο για να παρακολουθείτε τον λόγο για τον οποίο δημιουργήσατε έναν συγκεκριμένο κανόνα ή για να παρέχετε πλαίσιο για άλλους που ενδέχεται να δουν τους κανόνες σας.

## Μοτίβο URL

Υπάρχουν 3 επιλογές όπου μπορείτε να καθορίσετε ένα ή περισσότερα μοτίπα URL. *Ανακατεύθυνση Από*, *Εξαιρούμενα Πρότυπα URL* και *Επεξεργασία Ομάδας Σύλληψης*. Για να τις καθορίσετε, μπορείτε να επιλέξετε είτε **Wildcard** είτε **Κανονική Έκφραση**.

### Wildcard

Το **Wildcard** είναι ένας απλούστερος τύπος μοτίβου που σας επιτρέπει να χρησιμοποιείτε τα `*` (ταιριάζει με οτιδήποτε) και `?` (ταιριάζει με οποιονδήποτε μεμονωμένο χαρακτήρα) ως wildcards. Ακολουθούν μερικά παραδείγματα:

*   Για να ταιριάξετε το `https://example.com/hello`, μπορείτε να χρησιμοποιήσετε το `https://example.com/*`. Αυτό θα ταιριάξει με οποιαδήποτε συμβολοσειρά μετά το `https://example.com/`.
*   Για να ταιριάξετε το `https://example.com/search?q=hello`, μπορείτε να χρησιμοποιήσετε το `https://example.com/search?q=*`. Αυτό θα ταιριάξει με οποιαδήποτε τιμή για την παράμετρο `q`.
*   Για να ταιριάξετε οποιαδήποτε διεύθυνση URL που περιέχει τη λέξη `blog`, μπορείτε να χρησιμοποιήσετε το `*blog*`.

Μπορείτε επίσης να χρησιμοποιήσετε αντικατάσταση σε Wildcard, πράγμα που σημαίνει ότι μπορείτε να αναφερθείτε σε τμήματα της αντιστοιχισμένης διεύθυνσης URL χρησιμοποιώντας τα `$1`, `$2`, κ.λπ. Για παράδειγμα, αν χρησιμοποιήσετε `https://example.com/*-world-*`, και η διεύθυνση URL είναι `https://example.com/hello-world-goodbye`, τότε το `$1` θα ήταν "hello" και το `$2` θα ήταν "goodbye". Το `$0` είναι επίσης διαθέσιμο για αναφορά σε ολόκληρη την αντιστοιχισμένη διεύθυνση URL.

> [!Note]
> Αν και η αντικατάσταση είναι μια λειτουργία της Κανονικής Έκφρασης, μπορείτε να τη χρησιμοποιήσετε και για το Wildcard επειδή το Redirect Web μετατρέπει εσωτερικά το Wildcard σε Κανονική Έκφραση.

### Κανονική Έκφραση

Η **Κανονική Έκφραση** (Regex) είναι ένα ισχυρό εργαλείο για την αντιστοίχιση μοτίβων σε κείμενο, που υποστηρίζεται από τη μηχανή κανονικών εκφράσεων της Apple, όπως περιγράφεται [εδώ](https://developer.apple.com/documentation/foundation/nsregularexpression#1661042). Σας επιτρέπει να ορίσετε ένα συγκεκριμένο μοτίβο που αντιστοιχεί σε ένα σύνολο συμβολοσειρών. Ακολουθούν μερικά παραδείγματα.

-   Για να ταιριάξετε το `https://example.com/hello`, μπορείτε να χρησιμοποιήσετε το `https://example.com/(.*)`. Αυτό θα ταιριάξει με οποιαδήποτε συμβολοσειρά μετά το `https://example.com/` και θα την αποθηκεύσει σε μια ομάδα σύλληψης.
-   Για να ταιριάξετε το `https://example.com/search?q=hello`, μπορείτε να χρησιμοποιήσετε το `https://example.com/search\?q=(.*)`. Αυτό θα ταιριάξει με την τιμή της παραμέτρου `q` και θα την αποθηκεύσει σε μια ομάδα σύλληψης.
-   Για να ταιριάξετε οποιαδήποτε διεύθυνση URL που περιέχει τη λέξη `blog`, μπορείτε να χρησιμοποιήσετε το `.*blog.*`.

Μπορείτε να αναφερθείτε στις ομάδες σύλληψης στην *Ανακατεύθυνση Προς* ή στην *Αντικατάσταση* χρησιμοποιώντας τα `$1`, `$2`, ... ή το `$0` για αναφορά σε ολόκληρη την αντιστοιχία.

Περισσότερες πληροφορίες σχετικά με τη σύνταξη του Regex μπορείτε να βρείτε σε πηγές όπως το [RegExr](https://regexr.com/).

# Vianmääritys

Tällä sivulla luetellaan yleisiä ongelmia ja niiden ratkaisuja Redirect Webin käytössä.

## Uudelleenohjaussäännöt eivät toimi

Jos sääntösi eivät toimi odotetusti, tarkista seuraavat asiat:

- **Selaimen laajennus on asennettu ja käytössä oikealla sivulla:** Varmista, että olet asentanut Redirect Web -selainlaajennuksen ja että se on käytössä selaimessasi sillä verkkosivulla, jolta haluat ohjata uudelleen. Jos voit avata laajennuksen ponnahdusikkunan tällä tavalla lähdeverkkosivulla, se on asennettu oikein:
    <Image src={ExtensionPopup} alt="Redirect Web -laajennuksen ponnahdusikkuna" width="260" hasShadow/>

- **Säännön määritys on oikein:** Tarkista uudelleenohjaussääntösi Redirect Web -natiivisovelluksessa. Varmista, että lähde-URL-mallit, kohde-URL-osoitteet ja kaikki ehdot on asetettu oikein. Tarkistaaksesi, onko sääntösi kelvollinen, syötä esimerkkiosoite Esimerkit-osioon Muokkaa sääntöä -näytössä. Jos sääntösi on kelvollinen, näet odotetun kohde-URL-osoitteen syötteesi alla, näin:
    <Image src={RuleEditExampleResult} alt="Esimerkki säännön muokkausnäytöstä, joka näyttää esimerkkiosoitteen tuloksen" width="260" hasShadow/>

## Laajennuksen ponnahdusikkunassa näkyy virhe

<Image src={ErrorOverlay} alt="Redirect Web -virhepeite" width="260" hasShadow hasRoundedCorners/>

> Redirect Web -sovellusta ei ole asennettu tai se ei toimi oikein.  
> Hanki sovellus App Storesta.

Jos näet yllä olevan virheen avatessasi laajennuksen ponnahdusikkunan, se tarkoittaa, että Redirect Web -natiivisovellusta ei ole asennettu tai se ei toimi oikein.

Varmista, että olet asentanut uusimman version Redirect Webistä [App Storesta](https://apps.apple.com/app/id1571283503).

Saatat joutua käynnistämään selaimesi uudelleen natiivisovelluksen asentamisen jälkeen.

> [!NOTE]
> * Sinun on asennettava Redirect Web -natiivisovellus käyttääksesi selainlaajennusta. Laajennus yksinään ei toimi.
> * Safari-käyttäjien ei tarvitse asentaa laajennusta erikseen, koska se toimitetaan natiivisovelluksen mukana.

## Alkuperäinen sivu välähtää hetken ennen uudelleenohjausta

Tämä tapahtuu, kun käytät **alkuperäisen** tyyppisiä uudelleenohjaussääntöjä. Tämä tyyppi tarjoaa erilaisia edistyneitä ominaisuuksia, mutta se voi aiheuttaa lyhyen välähdyksen alkuperäisestä sivusta, koska se voi sallia alkuperäisen sivun latautumisen ennen uudelleenohjauksen suorittamista.

Avaa Muokkaa sääntöä -näyttö ja muuta **Tyyppi** **DNR**:ksi välttääksesi tämän ongelman.

<Image src={EditRuleDnr} alt="Redirect Web -laajennuksen ponnahdusikkuna" height="260" hasShadow hasRoundedCorners/>

> [!NOTE]
> Firefoxissa tätä ongelmaa ei pitäisi esiintyä edes alkuperäisen tyypin kanssa, koska se perustuu WebRequest API:iin, joka estää alkuperäisen sivun latautumisen.

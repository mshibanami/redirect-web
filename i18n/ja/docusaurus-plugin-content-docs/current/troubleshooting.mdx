# トラブルシューティング

このページでは、Redirect Web の使用時に発生する一般的な問題とその解決策を説明します。

<TOCInline toc={toc} />

## リダイレクトルールが機能しない

ルールが期待どおりに機能しない場合は、以下を確認してください。

- **ブラウザ拡張機能がインストールされ、正しいページで有効になっていること:** Redirect Web ブラウザ拡張機能がインストールされており、リダイレクト元の Web ページでブラウザで有効になっていることを確認してください。ソース Web ページでこのように拡張機能のポップアップを開くことができれば、正しくインストールされています。
    <Image src={ExtensionPopup} alt="Redirect Web extension popup" width="260" hasShadow/>

- **ルール設定が正しいこと:** Redirect Web ネイティブアプリでリダイレクトルールを再確認してください。ソース URL パターン、ターゲット URL、および条件が正しく設定されていることを確認してください。ルールが有効かどうかを確認するには、ルール編集画面の「Examples」セクションに例の URL を入力します。ルールが有効な場合は、入力の下に期待される宛先 URL が表示されます。
    <Image src={RuleEditExampleResult} alt="An example of rule edit screen showing the result of example URL" width="260" hasShadow/>

## 拡張機能のポップアップにエラーが表示される

<Image src={ErrorOverlay} alt="Redirect Web error overlay" width="260" hasShadow hasRoundedCorners/>

> Redirect Web アプリがインストールされていないか、正しく動作していません。
> App Store からアプリを入手してください。

拡張機能のポップアップを開いたときに上記のエラーが表示される場合は、Redirect Web ネイティブアプリがインストールされていないか、正しく実行されていないことを意味します。

[App Store](https://apps.apple.com/app/id1571283503) から Redirect Web の最新バージョンがインストールされていることを確認してください。

ネイティブアプリのインストール後にブラウザを再起動する必要がある場合もあります。

> [!NOTE]
> * ブラウザ拡張機能を使用するには、Redirect Web ネイティブアプリをインストールする必要があります。拡張機能だけでは機能しません。
> * Safari ユーザーは、ネイティブアプリにバンドルされているため、拡張機能を別途インストールする必要はありません。

## リダイレクト前に元のページが briefly 表示される

これは、**Original** タイプのリダイレクトルールを使用している場合に発生します。このタイプはさまざまな高度な機能を提供しますが、リダイレクトを実行する前に元のページが読み込まれる可能性があるため、元のページが briefly 表示されることがあります。

この問題を回避するには、ルール編集画面を開き、**Type** を **DNR** に変更します。

<Image src={EditRuleDnr} alt="Redirect Web extension popup" height="260" hasShadow hasRoundedCorners/>

> [!NOTE]
> Firefox では、WebRequest API に依存しているため、Original タイプでもこの問題は発生しません。これにより、元のページが読み込まれるのを防ぎます。

# Rozwiązywanie problemów

Ta strona zawiera listę typowych problemów i ich rozwiązań podczas korzystania z Redirect Web.

<TOCInline toc={toc} />

## Reguły przekierowania nie działają

Jeśli Twoje reguły nie działają zgodnie z oczekiwaniami, sprawdź następujące kwestie:

- **Rozszerzenie przeglądarki jest zainstalowane i włączone dla odpowiedniej strony:** Upewnij się, że zainstalowałeś rozszerzenie przeglądarki Redirect Web i że jest ono włączone w Twojej przeglądarce dla strony internetowej, z której chcesz przekierować. Jeśli możesz otworzyć wyskakujące okienko rozszerzenia w ten sposób na źródłowej stronie internetowej, jest ono zainstalowane poprawnie:
    <Image src={ExtensionPopup} alt="Wyskakujące okienko rozszerzenia Redirect Web" width="260" hasShadow/>

- **Konfiguracja reguły jest poprawna:** Dokładnie sprawdź swoje reguły przekierowania w natywnej aplikacji Redirect Web. Upewnij się, że wzorce źródłowych adresów URL, docelowe adresy URL i wszelkie warunki są poprawnie ustawione. Aby sprawdzić, czy Twoja reguła jest prawidłowa, wprowadź przykładowy adres URL w sekcji Przykłady na ekranie Edytuj regułę. Jeśli Twoja reguła jest prawidłowa, zobaczysz oczekiwany docelowy adres URL pod swoim wpisem, tak jak tutaj:
    <Image src={RuleEditExampleResult} alt="Przykład ekranu edycji reguły pokazujący wynik przykładowego adresu URL" width="260" hasShadow/>

## Wyskakujące okienko rozszerzenia pokazuje błąd

<Image src={ErrorOverlay} alt="Nakładka błędu Redirect Web" width="260" hasShadow hasRoundedCorners/>

> Aplikacja Redirect Web nie została zainstalowana lub nie działa poprawnie.
> Pobierz aplikację z App Store.

Jeśli widzisz powyższy błąd podczas otwierania wyskakującego okienka rozszerzenia, oznacza to, że natywna aplikacja Redirect Web nie jest zainstalowana lub nie działa poprawnie.

Upewnij się, że zainstalowałeś najnowszą wersję Redirect Web z [App Store](https://apps.apple.com/app/id1571283503).

Może być również konieczne ponowne uruchomienie przeglądarki po zainstalowaniu natywnej aplikacji.

> [!NOTE]
> * Musisz zainstalować natywną aplikację Redirect Web, aby korzystać z rozszerzenia przeglądarki. Samo rozszerzenie nie działa.
> * Użytkownicy Safari nie muszą instalować rozszerzenia osobno, ponieważ jest ono dołączone do natywnej aplikacji.

## Oryginalna strona miga krótko przed przekierowaniem

Dzieje się tak, gdy używasz reguł przekierowania typu **Oryginalny**. Ten typ zapewnia różne zaawansowane funkcje, ale może powodować krótkie mignięcie oryginalnej strony, ponieważ może pozwolić na załadowanie oryginalnej strony przed wykonaniem przekierowania.

Otwórz ekran Edytuj regułę i zmień **Typ** na **DNR**, aby uniknąć tego problemu.

<Image src={EditRuleDnr} alt="Wyskakujące okienko rozszerzenia Redirect Web" height="260" hasShadow hasRoundedCorners/>

> [!NOTE]
> W Firefoksie ten problem nie powinien występować nawet w przypadku typu Oryginalny, ponieważ opiera się on na API WebRequest, które zapobiega ładowaniu oryginalnej strony.

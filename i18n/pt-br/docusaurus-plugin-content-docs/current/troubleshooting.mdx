## As regras de redirecionamento não estão funcionando

Se suas regras não estão funcionando como esperado, verifique o seguinte:

- **A extensão do navegador está instalada e ativada para a página correta:** Certifique-se de ter instalado a extensão do navegador Redirect Web e que ela esteja ativada em seu navegador para a página da web da qual você deseja redirecionar. Se você conseguir abrir o pop-up da extensão como este na página da web de origem, ela está instalada corretamente:
    <Image src={ExtensionPopup} alt="Pop-up da extensão Redirect Web" width="260" hasShadow/>

- **A configuração da regra está correta:** Verifique novamente suas regras de redirecionamento no aplicativo nativo Redirect Web. Certifique-se de que os padrões de URL de origem, URLs de destino e quaisquer condições estejam configurados corretamente. Para verificar se sua regra é válida ou não, insira um URL de exemplo na seção Exemplos na tela Editar Regra. Se sua regra for válida, você verá o URL de destino esperado abaixo de sua entrada, assim:
    <Image src={RuleEditExampleResult} alt="Um exemplo de tela de edição de regra mostrando o resultado do URL de exemplo" width="260" hasShadow/>

## O pop-up da extensão mostra um erro

<Image src={ErrorOverlay} alt="Sobreposição de erro do Redirect Web" width="260" hasShadow hasRoundedCorners/>

> O aplicativo Redirect Web não foi instalado ou não está funcionando corretamente.  
> Por favor, obtenha o aplicativo na App Store.

Se você vir o erro acima ao abrir o pop-up da extensão, significa que o aplicativo nativo Redirect Web não está instalado ou não está funcionando corretamente.

Certifique-se de ter instalado a versão mais recente do Redirect Web na [App Store](https://apps.apple.com/app/id1571283503).

Você também pode precisar reiniciar seu navegador após instalar o aplicativo nativo.

> [!NOTE]
> * Você precisa instalar o aplicativo nativo Redirect Web para usar a extensão do navegador. A extensão sozinha não funciona.
> * Usuários do Safari não precisam instalar a extensão separadamente, pois ela é fornecida com o aplicativo nativo.

## A página original pisca brevemente antes do redirecionamento

Isso acontece quando você usa as regras de redirecionamento do tipo **Original**. Este tipo oferece vários recursos avançados, mas pode causar um breve piscar da página original porque pode permitir que a página original seja carregada antes de realizar o redirecionamento.

Abra a tela Editar Regra e altere o **Tipo** para **DNR** para evitar este problema.

<Image src={EditRuleDnr} alt="Pop-up da extensão Redirect Web" height="260" hasShadow hasRoundedCorners/>

> [!NOTE]
> No Firefox, este problema não deve ocorrer mesmo com o tipo Original, pois ele depende da API WebRequest, que impede que a página original seja carregada.

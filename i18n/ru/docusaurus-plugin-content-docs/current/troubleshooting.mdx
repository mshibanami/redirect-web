# Устранение неполадок

На этой странице перечислены распространённые проблемы и способы их решения при использовании Redirect Web.

<TOCInline toc={toc} />

## Правила перенаправления не работают {#rules-not-working}

Если ваши правила не срабатывают как ожидается, проверьте следующее:

- **Расширение для браузера установлено и включено для нужной страницы:** Убедитесь, что вы установили расширение Redirect Web и что оно включено в вашем браузере для той страницы, с которой вы хотите перенаправлять. Если на исходной странице вы можете открыть всплывающее окно расширения, как показано ниже, расширение установлено правильно:
    <Image src={ExtensionPopup} alt="Всплывающее окно расширения Redirect Web" width="260" hasShadow/>

- **Конфигурация правила корректна:** Проверьте правила перенаправления в нативном приложении Redirect Web. Убедитесь, что шаблоны исходных URL, целевые URL и любые условия настроены верно. Чтобы проверить, действительно ли правило работает, введите пример URL в разделе Examples на экране Edit Rule. Если правило действительно корректно, под вашим вводом отобразится ожидаемый целевой URL, как на примере ниже:
    <Image src={RuleEditExampleResult} alt="Пример экрана редактирования правила, показывающий результат для примерного URL" width="260" hasShadow/>

## Во всплывающем окне расширения отображается ошибка {#error-shown-in-extension-popup}

<Image src={ErrorOverlay} alt="Наложение ошибки Redirect Web" width="260" hasShadow hasRoundedCorners/>

> Приложение Redirect Web не установлено или работает неправильно.  
> Пожалуйста, установите приложение из App Store.

Если вы видите указанную выше ошибку при открытии всплывающего окна расширения, это означает, что нативное приложение Redirect Web не установлено или работает некорректно.

Пожалуйста, убедитесь, что вы установили последнюю версию Redirect Web из [App Store](https://apps.apple.com/app/id1571283503).

Возможно, после установки нативного приложения потребуется перезапустить браузер.

> [!NOTE]
> * Чтобы использовать расширение для браузера, необходимо установить нативное приложение Redirect Web. Только расширение работать не будет.
> * Пользователям Safari не нужно устанавливать расширение отдельно — оно входит в состав нативного приложения.

## Исходная страница кратко отображается перед перенаправлением {#original-page-flash}

Это происходит при использовании правил перенаправления типа **Original**. Этот тип предоставляет дополнительные возможности, но может вызвать краткое отображение исходной страницы, потому что исходная страница может загрузиться до выполнения перенаправления.

Откройте экран Edit Rule и измените **Type** на **DNR**, чтобы избежать этой проблемы.

<Image src={EditRuleDnr} alt="Всплывающее окно расширения Redirect Web" height="260" hasShadow hasRoundedCorners/>

> [!NOTE]
> В Firefox этой проблемы не должно возникать даже при типе Original, так как там используется API WebRequest, предотвращающий загрузку исходной страницы.

## В окне настроек Safari отображаются ошибки {#safari-style-loading-errors}

В окне настроек Safari вы можете увидеть сообщения об ошибках, похожие на следующие:

<Image src={PopupStyleLoadingError} alt="Скриншот сообщений об ошибках в окне настроек Safari" height="360" hasShadow hasRoundedCorners/>

> * Unable to find "popup.mobile.css" in the extension’s resources. It is an invalid path.
> * Unable to find "toolbar-button.mobile.css" in the extension’s resources. It is an invalid path.

Это не влияет на работу Redirect Web. Эти сообщения можно безопасно игнорировать.

Эти ошибки возникают потому, что всплывающее окно расширения пытается загрузить файлы стилей как для десктопной, так и для мобильной версии, тогда как может быть загружен только один из них в зависимости от того, используете ли вы macOS или iOS. В результате один из файлов всегда не сможет загрузиться — это ожидаемое поведение.

Мы понимаем, что сообщения об ошибках могут вызывать беспокойство, поэтому планируем подавить их в одном из будущих обновлений. Подробнее см. в [issue GitHub #96](https://github.com/mshibanami/redirect-web/issues/96). Извините за возникшую путаницу!

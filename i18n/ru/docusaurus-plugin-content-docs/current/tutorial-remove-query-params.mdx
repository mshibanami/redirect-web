# Случай 2: Удаление параметров запроса из URL

Предположим, в URL `example.com` есть параметр запроса `source=twitter`, и вы решили удалить его, чтобы анонимизировать себя.

В этом случае опция **Обработка захватываемых групп** является самым простым способом:

*   **Перенаправить с**: `https://example.com/*` (Wildcard)
*   **Перенаправить на**: `$0`
*   **Обработка захватываемых групп**:
    *   **Группа**: `$0`
    *   **Процесс**: Замена вхождений
        *   **Цель**: `&?source=[^&]*`
        *   **Замена**: *(нет)*
        *   **Шаблон текста**: Регулярное выражение

**<a href={useBaseUrl('rules/remove-parameters.redirectweb')} download>⬇️ Загрузить правило</a>**

Это правило работает следующим образом:

```
https://example.com/?source=twitter
➡️ https://example.com/?
```

```
https://example.com/?hello=world&source=twitter&foo=bar
➡️ https://example.com/?hello=world&foo=bar
```

Если вы хотите удалить больше параметров, добавьте больше процессов.

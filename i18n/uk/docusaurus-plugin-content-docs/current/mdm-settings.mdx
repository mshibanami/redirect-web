# Налаштування MDM для адміністраторів

RedirectWeb підтримує керовану конфігурацію додатків Apple (Managed App Configuration), що дозволяє вам (ІТ-адміністраторам) централізовано розгортати та керувати налаштуваннями додатка на пристроях співробітників/студентів за допомогою рішення для керування мобільними пристроями (MDM), такого як Jamf, Intune, Kandji тощо.

Ця сторінка описує ключі, що підлягають конфігурації, поведінку додатка, найкращі практики та обмеження.

## Огляд

> [!IMPORTANT]
>
> **Для розгортання через Apple Business Manager / Apple School Manager (VPP):**
> Стандартний додаток RedirectWeb використовує модель Freemium з покупками в додатку, яка не підходить для масових закупівель VPP. Щоб розгорнути повністю розблокований додаток у вашій організації, будь ласка, придбайте додаток **[RedirectWeb Pro](https://apps.apple.com/app/id6760209215)**, який є версією з повною попередньою оплатою, розробленою спеціально для розповсюдження через MDM/VPP.

Надсилаючи конфігурації через ваш MDM, ви можете досягти наступного:

- **Примусові правила:** Застосовуйте правила перенаправлення на рівні всієї організації, які користувачі не можуть змінювати або видаляти.
- **Початковий розподіл правил:** Розповсюджуйте набір правил-шаблонів, який слугує стандартним при першому запуску додатка.
- **Керування інтерфейсом:** Обмежуйте доступ до певних функцій (наприклад, Бібліотеки) та чітко вказуйте в додатку, що він перебуває під керуванням організації.
- **Отримання великих наборів правил (Fallback):** Обходьте обмеження розміру корисного навантаження MDM, отримуючи правила із зовнішнього сервера.

## Варіанти використання

- **Безперервна маршрутизація під час міграції систем:**
  Під час міграції зі застарілої внутрішньої системи (наприклад, `legacy-crm.internal.local`) на нову платформе (наприклад, `salesforce.com`), розповсюдьте `enforcedRuleSet`. Навіть якщо співробітники використовують старі закладки, вони будуть автоматично перенаправлені на нову систему, що усуває звернення до ІТ-підтримки щодо неактуальних посилань.
- **Допомога в доступі до інтрамережі:**
  Розповсюджуйте правила перенаправлення, використовуючи кастомні схеми URL, щоб полегшити доступ до ресурсів інтрамережі через VPN-тунелі для окремих додатків (Per-App VPN), такі як Omnissa Web.

> [!WARNING]
>
> Оновлення конфігурації MDM застосовуються негайно **лише якщо користувач відкриває додаток RedirectWeb**.
> Хоча додаток використовує фонові завдання для непомітного передавання оновлень MDM у розширення браузера, це виконання за принципом "найкращих зусиль", що диктується ОС. Час виконання є вкрай непередбачуваним, і завдання може не запуститися зовсім.
> Тому на цю функцію **не варто** покладатися для критично важливого реагування на інциденти (наприклад, спроба миттєво перенаправити всіх співробітників на резервний сайт протягом кількох хвилин після збою хмарного сервісу).

## Ключі конфігурації

Розгорніть наступні ключі та значення для Managed App Configuration через консоль MDM, зазвичай у форматі plist.

| Назва ключа            | Тип                              | Опис                                                                                                                                                                                                                                                                                                                           |
| ---------------------- | -------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `enforcedRuleSet`      | [Набір правил](#rule-set-object) | **[Примусові правила]** Набір правил, що застосовується з вищим пріоритетом, ніж звичайні правила. Перше правило у списку має перевагу. Користувачі не можуть редагувати, видаляти або змінювати порядок цих правил. Вони завжди активні, ігноруючи глобальний перемикач вимкнення додатка, і не синхронізуються через iCloud. |
| `prefilledRuleSet`     | [Набір правил](#rule-set-object) | **[Початкові правила]** Набір правил, що створюється замість стандартного прикладу правила, коли користувач запускає додаток вперше без наявних правил. Після розповсюдження вони діють як звичайні правила, дозволяючи користувачеві змінювати їх, видаляти та синхронізувати через iCloud.                                   |
| `organizationName`     | String                           | Відображає "Керується [organizationName]" у верхній частині екрана налаштувань у додатку, вказуючи користувачеві, що додаток керований.                                                                                                                                                                                        |
| `managedConfigVersion` | String                           | Довільний рядок версії для конфігурації. Відображається в інтерфейсі, дозволяючи ІТ-підтримці перевірити, чи надійшло останнє корисне навантаження MDM на пристрій під час усунення несправностей.                                                                                                                             |
| `showsLibrarySection`  | Boolean                          | Якщо встановлено `false`, функція "Бібліотека правил" прихована, а розділ замінюється повідомленням про помилку "Обмежено вашою організацією".                                                                                                                                                                                 |
| `fallback`             | [Fallback](#fallback-object)     | Вкажіть це, коли конфігурації перевищують ліміти корисного навантаження MDM. Додаток буде отримувати JSON/XML із цього URL для застосування конфігурації.                                                                                                                                                                      |

Ці ключі є необов'язковими.

### [Об'єкт] Набір правил {/* #rule-set-object */}

Кожен об'єкт правила має той самий формат, що й той, який ви можете [поширити/експортувати через додаток](export-or-import-rules), але вам потрібно конвертувати його з JSON у plist, ось так:

```xml
...
	<key>enforcedRuleSet</key>
	<dict>
		<key>kind</key>
		<string>RedirectList</string>
		<key>bundleID</key>
		<string>io.github.mshibanami.RedirectWebForSafari</string>
		<key>formatVersion</key>
		<string>5</string>
		<key>redirects</key>
		<array>
			<dict>
				<key>kind</key>
				<string>Redirect</string>
				<key>type</key>
				<string>originalRedirect</string>
				<key>title</key>
				<string>My enforced Rule</string>
				<key>sourceURLPattern</key>
				<dict>
					<key>type</key>
					<string>wildcard</string>
					<key>value</key>
					<string>https://example.com/1</string>
				</dict>
				<key>destinationURLPattern</key>
				<string>https://google.com/search?q=1</string>
			</dict>
		</array>
	</dict>
...
```

### [Об'єкт] Fallback {/* #fallback-object */}

| Назва ключа       | Тип          | Опис                                                                                                                                                          |
| ----------------- | ------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `fallbackDataURL` | String (URL) | URL-адреса кінцевої точки файлу JSON/XML, що містить набори правил.                                                                                           |
| `authToken`       | String       | Токен автентифікації, що додається до заголовка HTTP під час отримання даних із `fallbackDataURL`. (Використовується як `Authorization: Bearer [authToken]`). |

## Оцінка правил та вирішення конфліктів

1. Якщо надано `enforcedRuleSet`, його правила застосовуються з абсолютним пріоритетом.
2. Якщо нормальних правил немає, але існує `prefilledRuleSet`, `prefilledRuleSet` зберігається як нормальні правила.
3. Стандартне "Приклад правила перенаправлення" створюється лише тоді, коли правила MDM не налаштовані і нормальні правила відсутні.

## Поведінка Fallback URL

Коли ключ існує як у вбудованому корисному навантаженні MDM, так і в даних, отриманих через `fallbackDataURL`, додаток обробляє конфлікт наступним чином:

- **До завершення отримання:** Вбудовані налаштування суворо пріоритезуються та використовуються.
- **Після успішного отримання:** Дані, отримані з `fallbackDataURL`, перезаписують відповідні вбудовані налаштування.
- **Після невдалого отримання:** Додаток продовжує використовувати вбудовані налаштування.
- Незалежні ключі, які існують лише у вбудованому навантаженні, зберігаються.

## Зворотний зв'язок MDM та перевірка статусу

Ви можете перевірити, чи була конфігурація успішно застосована, через канал зворотного зв'язку MDM (`com.apple.feedback.managed`), якщо ваше рішення MDM підтримує його. Після обробки конфігурації (або завершення отримання даних за URL-адресою), додаток повертає наступні дані:

- `managedConfigVersion`: Рядок версії спроби конфігурації.
- `result`: Або `success`, або `error`.
- `messages`: Масив, що містить детальні повідомлення про помилки або попередження, якщо вони є.

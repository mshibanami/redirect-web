# Усунення несправностей

Ця сторінка містить список поширених проблем та їх вирішень під час використання Redirect Web.

<TOCInline toc={toc} />

## Правила перенаправлення не працюють

Якщо ваші правила не працюють належним чином, перевірте наступне:

- **Розширення браузера встановлено та увімкнено для правильної сторінки:** Переконайтеся, що ви встановили розширення браузера Redirect Web і що воно увімкнено у вашому браузері для веб-сторінки, з якої ви хочете перенаправити. Якщо ви можете відкрити спливаюче вікно розширення, як показано на вихідній веб-сторінці, воно встановлено правильно:
    <Image src={ExtensionPopup} alt="Redirect Web extension popup" width="260" hasShadow/>

- **Конфігурація правила правильна:** Перевірте свої правила перенаправлення в нативному додатку Redirect Web. Переконайтеся, що шаблони вихідних URL-адрес, цільові URL-адреси та будь-які умови налаштовані правильно. Щоб перевірити, чи є ваше правило дійсним, введіть приклад URL-адреси в розділі «Приклади» на екрані редагування правила. Якщо ваше правило дійсне, ви побачите очікувану URL-адресу призначення під вашим вводом, як показано тут:
    <Image src={RuleEditExampleResult} alt="An example of rule edit screen showing the result of example URL" width="260" hasShadow/>

## Спливаюче вікно розширення показує помилку

<Image src={ErrorOverlay} alt="Redirect Web error overlay" width="260" hasShadow hasRoundedCorners/>

> Додаток Redirect Web не встановлено або він працює неправильно.
> Будь ласка, завантажте додаток з App Store.

Якщо ви бачите вищезгадану помилку під час відкриття спливаючого вікна розширення, це означає, що нативний додаток Redirect Web не встановлено або він працює неправильно.

Будь ласка, переконайтеся, що ви встановили останню версію Redirect Web з [App Store](https://apps.apple.com/app/id1571283503).

Можливо, вам також знадобиться перезапустити браузер після встановлення нативного додатка.

> [!NOTE]
> * Вам потрібно встановити нативний додаток Redirect Web, щоб використовувати розширення браузера. Саме розширення не працює.
> * Користувачам Safari не потрібно встановлювати розширення окремо, оскільки воно входить до комплекту нативного додатка.

## Оригінальна сторінка коротко блимає перед перенаправленням

Це відбувається, коли ви використовуєте правила перенаправлення типу **Original**. Цей тип надає різні розширені функції, але він може спричинити короткочасне блимання оригінальної сторінки, оскільки він може дозволити оригінальній сторінці завантажитися перед виконанням перенаправлення.

Відкрийте екран редагування правила та змініть **Тип** на **DNR**, щоб уникнути цієї проблеми.

<Image src={EditRuleDnr} alt="Redirect Web extension popup" height="260" hasShadow hasRoundedCorners/>

> [!NOTE]
> У Firefox ця проблема не повинна виникати навіть з типом Original, оскільки він покладається на WebRequest API, який запобігає завантаженню оригінальної сторінки.
